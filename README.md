# AiTHOS — Cornell Athlete Health Passport (Demo)

Interactive, mobile-first demonstration of the **AiTHOS Athlete Health Passport** onboarding
experience, guided by the AiTHOS agent **SAiRA**. Built as a self-contained set of static
HTML pages — no build step, no dependencies, no backend.

> **Tagline:** *Your Turn. Your Terms.*
> **Entity:** Cadence Cove LLC — AiTHOS Division
> **Status:** Demonstration / validation prototype. Not for production use. See the disclaimer below.

---

## What's in this repo

| File | Purpose |
|------|---------|
| `index.html` | Entry point — a copy of the athlete sign-up page (so GitHub Pages serves it by default). |
| `AiTHOS_Athlete_Passport_Signup.html` | **Primary deliverable.** The SAiRA-guided athlete onboarding flow: live card assembly, sport/school selection, emergency contact, health goal, health-literacy module selection, date-of-birth/age check, in-page Health Fiduciary Agreement review & accept, parent hand-off, Apple Wallet animation, tap-to-enlarge card, and an in-page Health Literacy Modules overlay. |
| `AiTHOS_Parent_Landing.html` | Parent / guardian landing page. Personalizes from URL parameters passed by the athlete page (name, sport, year, school, pronoun, minor flag). Includes the SAiRA-guided completion flow and, for minors, a co-sign step with an in-page agreement modal. |
| `AiTHOS_Health_Modules.html` | Standalone Health Literacy Modules page (the same four modules are also embedded in the athlete page as an in-page overlay). |
| `assets/source-documents/` | Reference PDFs (demo fiduciary agreement, one-page brochure). **Not linked from the live pages** — the agreement is presented in-page as a review/accept modal. Kept for reference only. |

## How the pages connect

```
index.html  ─►  (same as)  AiTHOS_Athlete_Passport_Signup.html
                                   │
                                   │  at the end of onboarding, "Preview the
                                   │  message your parent receives →" links to:
                                   ▼
                            AiTHOS_Parent_Landing.html?athlete=…&team=…&year=…
                                   &school=…&pro=he|she|they&minor=0|1&parent=…

AiTHOS_Health_Modules.html  ─►  back link to  AiTHOS_Athlete_Passport_Signup.html
```

The athlete page passes data to the parent page through **URL query parameters**, so the
parent page renders the correct name, school, pronouns, and minor/adult flow. For these links
to resolve, **all HTML files must stay in the same directory** (which they are in this repo).

## Run it locally

No build step. Either:

- **Open directly:** double-click `index.html` (most features work from `file://`).
- **Serve locally (recommended)** so the athlete → parent link resolves cleanly:

  ```bash
  # Python 3
  python3 -m http.server 8000
  # then visit http://localhost:8000
  ```

## Deploy to GitHub Pages

1. Push this repo to GitHub.
2. Go to **Settings → Pages**.
3. Under **Build and deployment**, set **Source: Deploy from a branch**, branch **main**, folder **/ (root)**.
4. Save. Your site will be live at `https://<your-username>.github.io/<repo-name>/`.

`index.html` is the entry point and `.nojekyll` is included so GitHub Pages serves the files
as-is. Once live, the URL works on mobile and is safe to share for demos.

## Key features

- **SAiRA conversational onboarding** — warm, three-beat opening that frames health ownership as *stepping up, not stepping away*; an early privacy promise ("coaches and staff never see what's inside").
- **Live card assembly** — the metallic Ivy Health Passport builds in real time as the athlete answers; **tap the card to enlarge it** full-screen.
- **Sport & school dropdowns** — Men's / Women's Soccer; Business, CIS, Engineering, Human Ecology, ILR, Public Policy. Pronoun is inferred from sport for the parent page.
- **Health Literacy Modules** — four real modules (Know Your Health Identity; Show Up Ready for Care; Injury, Recovery & Return-to-Play; Own the Health Handoff), each with authentic content. SAiRA remembers the athlete's pick and, at the end of onboarding, offers to **deploy it now or later** with a real first-lesson preview (welcome, quiz, and Health Passport action). A "See all modules" overlay shows all four in-page.
- **Date-of-birth / age check** — drives the fiduciary structure: athletes 18+ sign as Principal; under-18 athletes require a parent/guardian co-signature.
- **In-page Health Fiduciary Agreement** — plain-language review & accept modal (no external PDF). Parent page shows a co-sign modal for minors.
- **Parent hand-off** — fully pronoun-aware; SAiRA guides the parent through adding providers and uploading records (and co-signing, for minors).

## Customization notes

The HTML files were generated by Python build scripts (not included here, as the output is
self-contained). To edit, work directly in the HTML/CSS/JS of each file. Things you may want to
swap for production:

- **Logos / trademarks.** The Cornell wordmark is a typographic placeholder and the centered
  card emblem uses a generic botanical ivy sprig — **not** the trademarked Cornell or Ivy League
  marks. Replace with licensed assets once you have rights.
- **Age of majority** is hard-coded at 18; it varies by U.S. state. Confirm with counsel.
- **Backend.** This is front-end only. SMS to parents, real Apple Wallet passes, record storage,
  a provider directory, and the live module engine all need to be wired up for production.
